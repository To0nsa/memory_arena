üéØ Introduction

Memory_Arena is a lightweight, high-performance memory allocator written in C, fully compatible with C++. Designed around the arena allocation model, it organizes memory allocations into contiguous blocks for rapid allocation and bulk deallocation, significantly improving speed, memory efficiency, and reducing fragmentation. Library that is especially useful for systems programming, game engines, real-time applications, and scenarios where performance and memory efficiency are critical.

Instead of individually managing numerous memory operations (like malloc and free), it allows you to reset or reuse entire memory regions at once, perfect for scenarios involving temporary data, real-time rendering, or modular systems.

‚ú® <tt>Memory_Arena</tt> Features

See Memory_Arena Features

‚ö° Fast Bump Allocation Allocations are performed linearly via pointer bumping, which offers near-zero overhead and avoids costly malloc bookkeeping. Ideal for temporary or scoped allocations.

üîß Flexible Allocation Allocate raw or zeroed memory, reallocate in place, or tag with debug labels. Supports alignment, hooks, stats, and thread safety. Designed for speed, clarity, and control‚Äîwithout manual frees or fragmentation.

üîÑ Dynamic Growth & Shrinkage Arenas can automatically grow and optionally shrink when memory usage changes. This provides flexibility while maintaining predictable performance characteristics.

üîÅ Sub-Arenas & Markers Supports nested memory scopes via sub-arenas and arena_mark/arena_pop. Useful for implementing undo/rollback systems or scoped temporary memory.

üßµ Thread-Safety (Opt-In) Enable safe multithreaded access to arena structures using ARENA_ENABLE_THREAD_SAFE. Internally guarded by recursive mutexes for safe reentry.

üåÄ Temporary Scratch Arenas Memory_Arena provides two systems for fast, reusable temporary memory: üî¢ Scratch Arena Pool (arena_scratch) Fast, reusable memory slots ideal for temporary workloads. Acquire/reset arenas on demand with thread-safe access, atomic tracking, and minimal overhead. Perfect for per-frame or per-task use. üßµ Thread-Local Scratch Arenas (arena_tlscratch) Each thread gets its own fast, auto-resetting arena. Zero locks, zero setup after init, and perfect for throwaway allocations in tight loops or parallel workloads.

üóÇÔ∏è Scoped Stack Frames Use arena_mark() and arena_pop() to create scoped memory lifetimes within an arena. Perfect for recursive algorithms, temporary parse buffers, or structured rollback. Fast, deterministic, and zero heap allocations‚Äîstack frames live inside the arena itself.

üíæ Arena Snapshots Save and load arena memory to .bin files. Includes magic header/versioning, offset tracking, and buffer content. Great for debugging, state persistence, or fast startup by restoring memory from disk. Only works with arenas that own their buffer.

üñºÔ∏è Interactive Memory Visualizer A curses-based terminal visualizer is provided (using Notcurses) to observe arena activity live, allocations, resets, growth, and more. Helpful for profiling, education, or debugging.

üß© Allocation Labels & Hooks Attach labels to allocations for better diagnostics, or use custom hooks to monitor memory usage and capture metadata in real time.

üìä Debug Stats & Growth Tracking Internal statistics provide detailed insight into memory usage, peak allocations, growth events, and frame stack depth.

üõ†Ô∏è Debug-Ready Built-in support for AddressSanitizer, ThreadSanitizer, and debug stats through CMake presets.

‚ò£Ô∏è Memory Poisoning for Debugging Enable ARENA_POISON_MEMORY to fill freed memory with a poison pattern, helping you catch use-after-free bugs during development.

üß™ Fully Tested Includes over 30 unit and multithreaded tests. Supports ASAN, TSAN, and custom debug checks with arena_report_error() on critical paths.

ü™∂ Minimal Dependencies Written in pure C11 with no external dependencies outside optional visualizer.

üß™ Debugging & Sanitizer Options

See Debugging & Sanitizer Options

To help catch memory leaks, bugs, race conditions, deadlocks early, this project includes optional sanitizer support and debug features built into the CMake configuration.

You can enable the following features by setting CMake options (using presets or -D flags):

  • **USE_ADDRESS_SANITIZER** ‚Äî Detects memory errors like use-after-free, buffer overflows, etc.
  • **USE_THREAD_SANITIZER** ‚Äî Detects data races in multi-threaded programs.
  • **ARENA_DEBUG_CHECKS** ‚Äî Adds runtime checks to verify internal consistency.
  • **ARENA_POISON_MEMORY** ‚Äî Overwrites memory with a known pattern when freed (e.g. 0xDEADBEEF) to detect use-after-free.
  • **ARENA_ENABLE_THREAD_SAFE** ‚Äî Enables mutex locking inside the allocator for safe multi-threaded usage.

‍‚ö†Ô∏è You cannot enable both ASAN and TSAN at the same time ‚Äî the build will fail with a clear error if you try.

These options are defined in:

  • SanitizerConfig.cmake ‚Äî applied conditionally based on flags
  • The top-level CMakeLists.txt ‚Äî integrates them into the build and install process

By using these, you can easily switch between production-ready builds and debug-friendly builds to help test your project more safely.

üìö Using The Libraries

See Using The Libraries

üîó Using the Static Library

The static version (libMemory_Arena.a) is linked directly into your binary. It requires no special setup at runtime ‚Äî ideal for portable or standalone builds.

target_link_libraries(my_app PRIVATE Memory_Arena::static)

This will include the memory arena directly into your binary, making it easy to distribute without worrying about runtime dependencies.

üì¶ Using the Shared Library

The shared version (libMemory_Arena.so) is loaded by the system at runtime. You must ensure the dynamic linker can locate it.

‚úÖ Option 1: Install Globally

sudo cmake --install build/release

This installs the .so to /usr/local/lib, which is typically already searched by the system loader.

  • ‚úÖ Pros: Works out of the box once installed, system-wide availability.
  • ‚ö†Ô∏è Cons: Requires sudo, may conflict with system packages if multiple versions are installed.

‚úÖ Option 2: Set <tt>LD_LIBRARY_PATH</tt>

export LD_LIBRARY_PATH=/path/to/build/release:$LD_LIBRARY_PATH
./my_app

Use this if you want to test without global installation. The environment variable tells the loader where to find your .so.

  • ‚úÖ Pros: Great for local testing, sandboxed environments, CI setups.
  • ‚ö†Ô∏è Cons: Needs to be set each time, or exported in .bashrc/.zshrc.

‍üí° Tip: You can also wrap it in a one-liner: LD_LIBRARY_PATH=build/release ./my_app

‚úÖ Option 3: Embed an rpath in Your Binary

set_target_properties(my_app PROPERTIES
INSTALL_RPATH "$ORIGIN/../lib"
)

This embeds a path inside your binary, instructing it to look for the shared library relative to its install location.

  • ‚úÖ Pros: Self-contained, portable, no need for global installation or env variables.
  • üí° $ORIGIN refers to the location of the executable at runtime ‚Äî useful when installing to a subfolder layout like:
my_app/

üìÇ Consuming the Library

See Consuming the Library

This project installs both CMake and pkg-config integration to simplify usage.

üîß From CMake

If you've installed the library globally (via sudo cmake --install), you can use find_package:

find_package(Memory_Arena REQUIRED)
# Link to either the static or shared version
target_link_libraries(my_app PRIVATE Memory_Arena::static) # or Memory_Arena::shared

You get access to:

  • Include paths via target_include_directories
  • Compile definitions like ARENA_ENABLE_THREAD_SAFE

üí° CMake will locate Memory_ArenaTargets.cmake and automatically configure everything.

üì¶ From pkg-config

If you're using a Makefile or a non-CMake build system:

pkg-config --cflags Memory_Arena # Get compiler flags
pkg-config --libs Memory_Arena # Get linker flags

Make sure PKG_CONFIG_PATH includes the install location (/usr/local/lib/pkgconfig).

export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH

This allows any build system to integrate with Memory_Arena using the standard pkg-config interface.
