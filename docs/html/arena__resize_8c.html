Detailed Description

Arena memory resizing logic — dynamic growth and shrinking.

Author
Toonsa
Date
2025

This file implements the dynamic memory resizing features of the arena allocator, allowing it to grow when allocations exceed current capacity, and shrink when memory usage drops significantly.

Features covered in this file:

  • Manual growth (arena_grow) based on requested size.
  • Automatic resizing policy through user-defined or default callbacks.
  • Shrinking (arena_shrink) of underused memory regions.
  • Heuristic-based auto-shrinking (arena_might_shrink) with safe thresholds.
  • Internal helpers for validation, computation, and buffer reallocation.

All operations are thread-safe and acquire internal locks on the arena during mutation of memory size or related metadata.

This module is especially useful for long-lived arenas in high-uptime applications, where memory usage patterns may fluctuate and reclaiming unused memory is desirable.

Note
The resizing logic assumes the arena owns its buffer and is marked as growable. Shrinking operations are designed to be conservative to prevent thrashing.
